# Introduction

Cells continually rewire metabolism in response to nutrient availability, tissue context, and stress. In cancer and development alike, these shifts are not just background biochemistry: they are part of how cells survive, proliferate, and adapt. Metabolomics should therefore be able to do more than list compounds. It should help describe how the metabolome changes when cells move into new conditions, and it should do so in a way that remains biologically interpretable. The published MetaboLiteLearner manuscript, archived in [source_papers/MetaboLiteLearner_published_paper.pdf](source_papers/MetaboLiteLearner_published_paper.pdf), was written to address exactly that gap by treating fragmentation spectra as predictive objects rather than as annotation leftovers.

The standard metabolomics split between targeted and untargeted analysis is useful but limiting. Targeted workflows are accurate for a predefined set of metabolites, yet they can miss unexpected or unidentified compounds. Untargeted workflows broaden coverage, but they still face reproducibility problems, unknown-identity bottlenecks, and a tendency to stop at detection rather than explanation. The 2024 paper argues that fragmentation spectra already preserve information about molecular structure and that this information can be used before full metabolite identification is complete. In that framing, the learning problem becomes: can the spectral features of a metabolite predict how its abundance changes when cells undergo a particular rewiring program?

The original answer was a small but explicit yes. MetaboLiteLearner uses electron ionization spectra collected in GC/MS scan mode, bins them into a 550-dimensional representation, and fits a partial least squares regression model to predict log2 fold changes in brain-homing and lung-homing derivatives of the MDA-MB-231 breast cancer line. The published study reported a MetaboLiteLearner Open Dataset of 153 unique spectra and recovered shared as well as lineage-specific rewiring signals without requiring prior metabolite identification or a curated metabolic network. This repository carries a current Python baseline figure set in [../data/original_study/folds/figure_1.png](../data/original_study/folds/figure_1.png), [../data/original_study/folds/figure_3.png](../data/original_study/folds/figure_3.png), and [../data/original_study/folds/loadings.png](../data/original_study/folds/loadings.png), together with the workflow implementation in [../metabolite_learner/workflow.py](../metabolite_learner/workflow.py).

MetaboLiteLearner 2.0 should keep that claim set intact while tightening the manuscript around it. The point is not to inflate the original result into a broader claim than the data support, but to restate the 2024 paper cleanly, make the representation and evaluation choices easier to follow, and prepare the ground for stronger benchmarks, clearer failure analysis, and better separation between what was shown in the published study and what this successor workspace intends to improve.
